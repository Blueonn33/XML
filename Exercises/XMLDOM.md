# XMLDOM - Въведение

## Какво е DOM

DOM определя стандарт за достъп и манипулиране на документи. DOM е интерфейс, който позволява на програми и скриптове динамично да осъществяват достъп и да актуализират съдържанието, структурата и стила на документ.

XML DOM определя стандартен начин за достъп и манипулиране на XML документи. Той представя XML документа като дървовидна структура

## XML DOM

Всички XML елементи са достъпни чрез XML DOM. DOM е:
- стандартен обектен модел за XML
- програмен интерфейс за XML
- независим от платформата и езика
- стандарт на W3C

### Извличане на стойността на XML елемент

Кодът извлича текстовата стойност на първия елемент <title> в XML документ

```js
txt = xmlDoc.getElementsByTagName("title")[0].childNodes[0].nodeValue;
```

### Зареждане на XML низ

Примерът зарежда текстов низ в XML DOM обект и извлича информацията от него с JavaScript

```html
<html>
<body>

<p id="demo"></p>

<script>
var text, parser, xmlDoc;

text = "<bookstore><book>" +
"<title>Everyday Italian</title>" +
"<author>Giada De Laurentiis</author>" +
"<year>2005</year>" +
"</book></bookstore>";

parser = new DOMParser();
xmlDoc = parser.parseFromString(text,"text/xml");

document.getElementById("demo").innerHTML =
xmlDoc.getElementsByTagName("title")[0].childNodes[0].nodeValue;
</script>

</body>
</html>
```

## Програмен интерфейс

DOM моделира XML като набор от обекти-възли. Достъпът до възлите може да се осъществи чрез JavaScript или други езици. 

Свойствата често се наричат nodename - book
Методите се наричат нещо, което е направено (изтриване на книга)

### XML DOM свойства

- x.nodeName - името на х
- x.nodeValue - стойността на х
- x.parentNode - родителският възел на х
- x.childNodes - дъщерните възли на х
- x.attributes - възлите на атрибутите на х

( х е от тип възел )

### XML DOM методи

- x.getElementsByTagName( name ) - получава всички елементи с посочено име на таг
- x.appendChild ( node ) - вмъква дъщерен възел към x
- x.removeChild ( node ) - премахва дъщерен възел 

( х е от тип възел )

# XML DOM възли

Според XML DOM, всичко в един XML документ е възел'
- целият документ е възел на документа
- всеки XML елемент е възел на елемент
- Текстът в XML елементите е текстови възли
- Всеки атрибут е атрибутен възел
- Коментарите са възли за коментари

## Пример за DOM

```xml
<?xml version="1.0" encoding="UTF-8"?>
<bookstore>
  <book category="cooking">
    <title lang="en">Everyday Italian</title>
    <author>Giada De Laurentiis</author>
    <year>2005</year>
    <price>30.00</price>
  </book>
  <book category="children">
    <title lang="en">Harry Potter</title>
    <author>J K. Rowling</author>
    <year>2005</year>
    <price>29.99</price>
  </book>
  <book category="web">
    <title lang="en">XQuery Kick Start</title>
    <author>James McGovern</author>
    <author>Per Bothner</author>
    <author>Kurt Cagle</author>
    <author>James Linn</author>
    <author>Vaidyanathan Nagarajan</author>
    <year>2003</year>
    <price>49.99</price>
  </book>
  <book category="web" cover="paperback">
    <title lang="en">Learning XML</title>
    <author>Erik T. Ray</author>
    <year>2003</year>
    <price>39.95</price>
  </book>
</bookstore>
```

- Коренният възел се нарича bookstore
- Всички останали възли в документа се съдържат в bookstore
- Коренният възел съдържа 4 възела
- Първият възел book съдържа дъщерните възли title, author, year, price
- Дъщерните възли съдържат по 1 текстов възел

**Текстът винаги се съхранява в текстови възли**

## Дървото на възлите

XML DOM разглежда XML документ като дървовидна структура. Дървовидната структура се нарича дърво-възел. Достъпът до всички възли е възможен през дървото. Съдържанието има може да бъде променяно или изтривано, както и да се създават нови елементи.

Дървото на възлите показва множеството от възли и връзките между тях. Дървото започва от коренния възел и се разклонява към текстовите възли на най-ниското ниво на дървото

### Родители, деца, братя и сестри на възли

Възлите в дървото на възлите имат йерархична връзка помежду си. Термините родител, дете и брат/сестра се използват за описание на взаимоотношенията. Родителските възли имат деца. Децата на едно и също ниво се наричат братя/сестри.

- в дърво на възлите, най-горният възел се нарича корен
- всеки възел, с изключение на коренния, има точно 1 родителски възел
- възел може да има произволен брой деца
- листът е възел без деца
- братя и сестри са възли с един и същ родител

Тъй като XML данните са структурирани във формата на дърво, те могат да бъдат прегледани, без да се знае точната структура на дървото и без да се знае типът данни, съдържащи се в него.

### Първо дете - последно дете

Елементът title е първия дъщерен елемент на елемента book, а елементът price е последния дъщерен елемент на елемента book. Освен това елементът book е родителски възел на елементите title, author, year, price

```xml
<bookstore>
  <book category="cooking">
    <title lang="en">Everyday Italian</title>
    <author>Giada De Laurentiis</author>
    <year>2005</year>
    <price>30.00</price>
  </book>
</bookstore>
```

# XML DOM - достъп до възли

С DOM можете да получите достъп до всеки възел в XML документ. Достъпът до възел може да бъде осъществен по 3 начина:
- Чрез използване на метода getElementByTagName()
- Чрез преминаване през дървото на възлите
- Чрез навигиране в дървото на възлите, използвайки връзките между възлите

### Методът getElementsByTagName()

Методът връща всички елементи с посочено име на таг

node.getElementsByTagName("tagname");

#### Примери

Връща само елементи title под възела x

x.getElementsByTagName("title");

За получаване на всички елементи title се използва xmlDoc (самия документ)

xmlDoc.getElementsByTagName("title");

## Списък с DOM възли

Методът getElementsByTagName връща списък с възли. Списъкът с възли е масив от възли.

`x.getElementsByTagName("title");` 

Елементите title могат да бъдат достъпни по индексен номер. За достъп до третия title се пише:

y = x[2]

### Дължина на списъка с DOM възли

Свойството length определя дължината на списък с възли (броя на възлите)

```js
var x = xmlDoc.getElementsByTagName("title");

for (i = 0; i < x.length; i++){
    ...
}
```

### Видове възли

- Свойството documentElement на XML документа е коренният възел. 
- Свойството nodeName на възел е името на възела. 
- Свойството nodeType на възел е типа на възела

### Преминаване през възли

Следният код преминава през дъщерните възли, които са също така елементни възли на коренния възел

```js
txt = "";
x = xmlDoc.documentElement.childNodes;

for (i = 0; i < x.length; i++){
    // Process only element nodes ( type 1 )

    if (x[i].nodeType == 1) {
        txt += x[i].nodeName + "<br>";
    }
}
```

- Заредили сме books.xml в xmlDoc.
- Получаваме дъщерните възли на коренния елемент (xmlDoc)
- За всеки дъщерен възел проверяваме типа на възела. Ако типът на възела е 1, той е елементен възел
- Изведете името на възела, ако е елементен възел

### Навигиране във взаимоотношенията между възлите

Кодът навигира в дървото на възлите, използвайки връзките между възлите

```js
x = xmlDoc.getElementsByTagName("book")[0];
xlen = x.childNodes.length;
y = x.firstChild;

txt = "";
for (i = 0; i < xlenl; i++){
    // Process only element nodes (type 1)
    if (y.nodeType == 1) {
        txt += y.nodeName + "<br>";
    }
    y = y.nextSibling;
}
```

# XML DOM информация за възлите

Свойствата nodeName, nodeValue и nodeType съдържат информация за възлите

## Свойства на възела

В XML DOM всеки възел е обект. Обектите имат методи и свойства, до които може да се осъществи достъп и да се манипулират чрез JavaScript. Три важни свойства на възела са:
- име на възела
- стойност на възела
- тип на възела

## Свойството nodeName

Свойството указва името на възел
- nodeName е само за четене
- nodeName на елемент node е същото като името на тага
- nodeName на атрибутен възел е името на атрибута
- nodeName на текстов възел винаги e #text
- nodeName на възела документ винаги е #document

## Свойството nodeValue

Указва стойността на възел
- nodeValue за възлите на елементите е неопределена
- nodeValue за текстови възли е самия текст
- nodeValue за атрибутни възли е стойността на атрибута

## Стойност на елемент

Кодът извлича стойността на текстовия възел на първия елемент title

var x = xmlDoc.getElementsByTagName("title")[0].childNodes[0];
var txt = x.nodeValue;

## Промяна на стойност на елемент

Следният код променя стойността на текстовия възел на първия елемент title

var x = xmlDoc.getElementsByTagName("title")[0].childNodes[0];
x.nodeValue = "Easy Cooking";

## Свойството nodeType

То определя типа на възела. Най-важните типове възли са:
- element
- attribute
- text
- comment
- document

# Списък с XML DOM възли

Списък с възли се връща от метода getElementsByTagName() и свойството childNodes

Когато се използват свойства или методи като childNodes или getElementsByTagName(), се връща обект от списък с възли. Обектът тип списък с възли представлява списък от възли, в същия ред, както в XML файла. Достъпът до възлите в списъка с възли се осъществява с индексни номера, започващи от 0

## Дължина на списъка с възли

Обектът тип списък с възли се самоактуализира. Ако даден елемент бъде изтрит или добавен, списъкът се актуализира автоматично. Свойството за дължина на списък с възли е броят на възлите в списъка

Този код връща броя на елементите title

```js
x = xmlDoc.getElementsByTagName("title").length;
```

## Списък с атрибути на DOM 

Свойството attributes на елементен възел връща списък с атрибутни възли. Това се нарича карта на именувани възли и е подобно на списък с възли, с изключение на някои разлики в методите и свойствата. 

Кодът връща списък с атрибутни възли от първия елемент book

x = xmlDoc.getElementsByTagName('book')[0].attributes

След изпълнение на горния код, x.length е броят на атрибутите, а x.getNamedItem() може да се използва за връщане на възел на атрибут

Кодът получава стойността на атрибута category и броя на атрибутите на дадена книга

x = xmlDoc.getElementsByTagName("book")[0].attributes
txt = x.getNamedItem("category").nodeValue + " " + x.length

# Дърво на възлите за преминаване през XML DOM

Преминаването означава пътуване през дървото на възлите. Често е необходимо да се премине през целия XML документ, за да се извлече стойността на всеки елемент. 

## Пример

Примерът преглежда всички дъщерни възли на book и показва техните имена и стойности

```html
<!DOCTYPE html>
<html>
  <body>
    <p id="demo"></p>

    <script>
      var x, i, xmlDoc;
      var txt = "";
      var text = 
        "<book>" + "<title>Italian</title>" + "<author>Giana</author>" + "<year>2004</year>" + "</book>";
        
      parser = new DOMParser();
      xmlDoc = parser.parseFromString(text, "text/xml");

      // documentElement always represents the root node
      x = xmlDoc.documentElement.childNodes;
      for (i = 0; i < x.length; i++){
        txt += x[i].nodeName + ":" + x[i]childNodes[0].nodeValue + "<br>";
      }

      document.getElementById("demo").innerHTML = txt;
    </script>
  </body>
</html>
```

## DOM - white spaces и нови редове

XML често съдържа символи за нов ред или интервал между възли. Това често се случва, когато документът се редактира от прост редактор като Notepad.

Някои браузъри НЕ третират празните интервали или новите редове като текстови възли, докато други браузъри го правят.

## PCDATA - анализират символни данни

XML парсерите обикновено анализират целия текст в XML документ. Когато XML елемент се анализира, текстът между XML таговете също се анализира.

Парсерът прави това, защото XML елементите могат да съдържат други елементи, както в този пример, където елементът name съдържа 2 други елемента

<name><first>Martin</first><last>Marinov</last></name>

Парсерът ще го раздели на поделементи

<name>
  <first>Martin</first>
  <last>Marinov</last>
</name>

Анализирани символни данни (PCDATA) е термин, използван за текстови данни, които ще бъдат анализирани от XML парсера

## CDATA - (неанализирани) данни за символи

Терминът CDATA се използва за текстови данни, които не трябва да бъдат анализирани от XML парсера.

Символи като "<" и "&" са недопустими в XML елементи
- < генерира грешка, защото парсерът го интерпретира като начало на нов елемент
- & генерира грешка, защото парсерът го интерпретира като начало на символен обект

За да се избегнат грешки, кодът се дефинира като CDATA

```html
<script>
  <![CDATA[
    function matchTwo(a,b){
      if (a < b && a < 0){
        return 1;
      }
      else {
        return 0;
      }
    }
  ]]>
</script>
```

# XML DOM - навигиране на възли

Възлите могат да бъдат навигирани с помощта на взаимоотношения между възлите. Достъпът до възли в дървото на възлите чрез връзката между възлите често се нарича **навигиране на възли**

В XML DOM, връзките между възлите са дефинирани като свойства на възлите:
- parentNode;
- childNodes;
- firstChild;
- lastChild;
- nextSibling;
- previousSibling;

Следното изображение илюстрира част от дървото на възлите и връзката между възлите

## DOM - parentNode

Всички възли имат точно 1 родителски възел. Следният код навигира към родителския възел на book

```js
function myFunction(xml) {
  var xmlDoc = xml.responseXML;
  var x = xmlDoc.getElementsByTagName('book')[0];
  document.getElementById("demo").innerHTML = x.parentNode.nodeName;
} 
```

# XML DOM - стойности на възли

Свойството nodeValue се използва за получаване на текстовата стойност на възел. Методът getAttribute() връща стойността на атрибут

## Получаване на стойността на елемент

В DOM всичко е възел. Възлите на елементите нямат текстова стойност. Текстовата стойност на елементен възел се съхранява в дъщерен възел. Този възел се нарича текстов възел.

:::note
За да се извлече текстовата стойност на елемент, трябва да се извлече стойността на текстовия възел на елемента.
:::

## Методът getElementsByTagName

Методът връща списък с възли на всички елементи, с посоченото име на тага, в същия ред, в който се появяват в изходния документ

Кодът извлича първия елемент title

var x = xmlDoc.getElementsByTagName("title")[0];

## Свойството childNodes

childNodes връща списък с дъщерните възли на елемента. Кодът извлича текстовия възел на първия елемент title

x = xmlDoc.getElementsByTagName("title")[0];
y = x.childNodes[0];

## Свойството nodeValue

Връща текстовата стойност на текстов възел. Кодът извлича текстовата стойност на текстовия възел на първия елемент title

x = xmlDoc.getElementsByTagName("title")[0];
y = x.childNodes[0];
z = y.nodeValue;

## Получаване на стойността на атрибут

В DOM атрибутите са възли. За разлика от елементите възли, атрибутните възли имат текстови стойности. Начинът за получаване на стойността на даден атрибут е като се получи неговата текстова стойност. Това може да се направи с помощта на метода getAttribute() или с помощта на свойството nodeValue на метода getAttributeNode()

### Стойност чрез getAttribute()

Методът извлича стойността на атрибут. Следният код извлича текстовата стойност на атрибута lang на първия елемент title

```js
const title = xmlDoc.getElementsByTagName("title")[0];
let txt = title.getAttribute("lang");
```
